내 GRU 금가격 예측 코드에서 데이터 누수 가능성을 줄이고 싶어.

현재는 TimeSeriesSplit으로 교차검증하고 있는데,
타겟이 "현재 시점 기준 60일 후 이동평균 수익률"이라서
train과 validation 사이에 미래 정보가 겹칠 수 있어.

다음 조건으로 코드를 수정해줘.

1. 시계열 데이터이므로 랜덤 셔플 금지
2. train과 validation 사이에 최소 60일 gap을 둘 것
3. 현재 seq_len=120, horizon=60 기준으로 데이터 누수가 없도록 split을 수정할 것
4. scaler를 쓴다면 train 데이터에만 fit하고 validation에는 transform만 할 것
5. 전체 실행 가능한 완성형 코드로 보여줄 것
6. 어디서 누수가 발생할 수 있었는지 주석으로 쉽게 설명할 것

내 기존 코드 구조를 최대한 유지해서 수정해줘.
